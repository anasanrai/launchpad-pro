# Paddle Setup Guide for LaunchPad Pro v2

## 1. Paddle API Keys (Sandbox & Live)

### Sandbox API Key
| Field | Value | Notes |
|-------|-------|-------|
| **Name** | LaunchPad Pro Sandbox | For testing in development |
| **Description** | Development API key for LaunchPad Pro testing environment | Used for testing payments, webhooks, and subscription flows |
| **Permissions** | `transactions:read`, `transactions:write`, `subscriptions:read`, `subscriptions:write`, `customers:read`, `customers:write`, `products:read`, `products:write`, `webhooks:read`, `webhooks:write` | Full access needed for development |
| **Environment** | Sandbox | Test environment |
| **Status** | Active | Must be enabled |

**Where to find:** Paddle Dashboard → Settings → API Keys → Create New Key → Select Sandbox

---

### Live API Key
| Field | Value | Notes |

## 2. Paddle Webhook Configuration

### Webhook Secret
| Field | Value | Notes |
|-------|-------|-------|
| **Description** | LaunchPad Pro Webhook Secret | Used to verify webhook signatures from Paddle |
| **URL** | `https://launchpad-pro.vercel.app/api/paddle/webhook` | Production webhook endpoint |
| **Sandbox URL** | `https://localhost:3000/api/paddle/webhook` | Local development endpoint (for testing) |
| **Status** | Active | Must be enabled |

### Events to Subscribe
| Event | Description | Action |
|-------|-------------|--------|
| `subscription.created` | New subscription created | Activate user subscription in database |
| `subscription.updated` | Subscription updated (plan change) | Update subscription tier in database |
| `subscription.activated` | Subscription activated after payment | Grant access to features |
| `subscription.canceled` | Subscription canceled by user | Downgrade user to free tier |
| `subscription.paused` | Subscription paused | Pause feature access |
| `subscription.resumed` | Subscription resumed | Restore feature access |
| `transaction.completed` | Payment completed | Update subscription status, send confirmation email |
| `transaction.billed` | Payment billed | Log transaction in database |
| `transaction.payment_failed` | Payment failed | Notify user, retry logic |

**Where to find:** Paddle Dashboard → Notifications → Webhooks → Create New Webhook

## Environment Variables to Add to Vercel

Once you have the credentials from Paddle, add these to your Vercel project settings:

```
PADDLE_API_KEY_SANDBOX=your_sandbox_api_key_here
PADDLE_API_KEY_LIVE=your_live_api_key_here
PADDLE_WEBHOOK_SECRET_SANDBOX=your_sandbox_webhook_secret_here
PADDLE_WEBHOOK_SECRET_LIVE=your_live_webhook_secret_here
PADDLE_PRODUCT_ID_STARTER=prod_xxxxxxxxxxxxx
PADDLE_PRODUCT_ID_PRO=prod_xxxxxxxxxxxxx
PADDLE_PRODUCT_ID_AGENCY=prod_xxxxxxxxxxxxx
PADDLE_ENVIRONMENT=sandbox  # Change to 'live' for production
```
